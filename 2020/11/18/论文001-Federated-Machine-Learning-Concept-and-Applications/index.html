联合机器学习: 概念及应用

2挑战: 数据孤岛, 数据隐私和安全.

我们提出了安全联合学习, 除了google_2016 我们引入了全面的联合学习框架:

  • 水平联合学习
  • 垂直联合学习
  • 转移联合学习

在组织间建立数据网络, 不伤害用户隐私的前提下共享知识

intro

2016 alphago 走向成熟, 但是现实:

紧缺的数据或者低质量的数据:

通过跨组织传输可以融合数据, 但是由隐私保护, 行业竞争等限制,

同时: 大公司的数据安全和隐私问题, 数据的交易程序也是新的挑战

so : 如何解决数据碎片化和孤岛问题是主要的挑战

我们调查了FL现有的工作, 希望重点从改善AI模型性能 到 符合数据隐私和安全法的集成方法上

FL 概述

google提出, 最近改进在统计挑战和安全性上.

也有让更加个性化FL的研究

现有集中在分布式移动设备的交互上: 优化重点:

  • 大规模分布的通信成本
  • 不平衡的数据分布
  • 设备可靠性

为了包括机构间合作学习, 我们扩展联邦学习到更通用

​ 隐私保护 分布式 合作 学习上

本文进一步调查了安全基础

FL的定义

N个拥有者 整合他们的数据, 传统是把所有数据放在一起去训练M_SUM

现在是一起训练M_FED, 而且不暴露他们的数据给其他人, 同时sum和fed的精确度要尽可能接近, δ-accuracy loss.

FL安全

这就需要安全模型和分析来提供有意义的隐私保证。

我们简要回顾和比较用于联合学习的不同隐私技术。我们还确定了防止间接泄漏（indirect leakage）的方法和潜在挑战。

差分隐私（Differential Privacy） k-anonymity

涉及给数据添加噪声，或使用归纳方法掩盖某些敏感属性，从而使数据无法恢复以保护用户隐私。

但是根源仍然要求将数据传输到其他地方，这通常需要在准确性和隐私之间进行权衡。
